# CLAUDE.md

本仓库用于沉淀 Claude Code 的项目协作方法。默认遵循以下规则：

## 1. 必读文件

执行任务前，优先阅读：

- `/docs/project_brief.md`
- `/docs/project_status.md`
- 与当前任务直接相关的源码或文档

不要无意义全仓扫描。

## 2. 工作原则

- 只做用户明确要求的事
- 先理解上下文，再开始修改
- 修改后运行相关测试或检查
- 输出保持简洁，避免长篇推理
- 对不确定信息明确标记“待确认”

## 3. 安全底线

未经确认，禁止：

- 删除核心文件
- 覆盖配置文件
- 修改生产环境配置
- 批量删除文件或数据
- 执行不可逆操作
- 引入明显费用或外部数据泄露风险

## 4. 允许自动执行

以下操作可直接执行：

- 新增普通文档
- 修复明确的文档问题
- 增补模板文件
- 小范围重构
- 更新状态记录

## 5. 任务结束要求

每次完成任务后，尽量补充：

- 修改文件列表
- 核心变更摘要
- 测试结果
- 遗留问题
- 下一步建议
