# Project Brief

## 1. 项目名称

- 名称：Claude Code 项目四件套实操配置指南

## 2. 项目一句话目标

帮助本地项目建立一套稳定的 Claude Code 协作基线，降低上下文丢失、误改和状态不一致的风险。

## 3. 使用者与场景

- 主要使用者：个人开发者、独立维护者
- 使用场景：本地 Python AI 项目、文档型项目、原型项目
- 使用频率：持续迭代期间高频使用
- 是否个人本地使用：是

## 4. 项目类型

- 本地 Python AI 项目协作指南
- 是否需要 CLI：不强依赖
- 是否需要 Web UI：否
- 是否需要数据库：否
- 是否需要调用外部模型 API：视项目而定
- 是否需要本地模型：视项目而定

## 5. 核心功能

### MVP 必须包含

1. 四件套概念说明
2. 可直接复用的模板
3. 清晰的任务执行顺序

### 后续可以加入

1. 更多任务模板
2. 不同项目类型的变体
3. 常见问题与排障清单

### 明确暂不做

1. 复杂自动化工作流
2. 依赖特定框架的深度绑定
3. 生产级平台化功能

## 6. 核心流程

1. 读取项目目标
2. 约束 Claude Code 行为
3. 记录当前状态
4. 按模板执行任务
5. 任务结束后更新状态

## 7. 技术约束

- Python 版本：待确认
- 包管理方式：待确认
- 主要框架：待确认
- 模型调用方式：待确认
- 数据存储方式：待确认
- 测试框架：待确认
- 不希望引入的依赖：待确认

## 8. 安全与成本约束

- 不上传敏感文件
- 不在日志中记录 API key
- 不默认执行高成本模型调用
- 不修改生产环境配置
- 不执行不可逆操作
- 不批量删除文件或数据

## 9. 验收标准

1. 读者能快速理解四件套分别解决什么问题
2. 读者能直接复用模板文件
3. 读者能把这套方法迁移到自己的项目里

## 10. 当前优先级

1. 保持说明清晰、可落地
2. 补齐可复用模板
3. 持续维护状态记录

## 11. 备注

本仓库是方法论和模板仓库，不绑定某个具体业务项目。
