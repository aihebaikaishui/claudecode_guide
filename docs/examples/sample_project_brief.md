# Project Brief - Sample

## 1. 项目名称

- 名称：本地文档问答助手

## 2. 项目一句话目标

构建一个本地运行的文档问答工具，支持读取指定文件夹中的 Markdown、PDF 和 TXT 文档，并基于这些内容回答问题。

## 3. 使用者与场景

- 主要使用者：个人知识工作者
- 使用场景：查阅项目资料、会议纪要、研究笔记
- 使用频率：每天多次
- 是否个人本地使用：是

## 4. 项目类型

- 本地 Python AI 项目
- 是否需要 CLI：是
- 是否需要 Web UI：后续考虑
- 是否需要数据库：否
- 是否需要调用外部模型 API：是
- 是否需要本地模型：可选

## 5. 核心功能

### MVP 必须包含

1. 扫描本地文档目录
2. 提取文本并建立可检索索引
3. 根据用户问题返回答案和引用来源

### 后续可以加入

1. Web UI
2. 增量索引
3. 多轮对话上下文

### 明确暂不做

1. 多用户协作
2. 云端同步
3. 复杂权限系统

## 6. 核心流程

1. 用户指定文档目录
2. 程序读取并切分文档
3. 程序建立索引
4. 用户提问
5. 程序返回答案和来源

## 7. 技术约束

- Python 版本：3.11+
- Python 包管理器和环境管理工具：uv
- 主要框架：FastAPI, LangChain
- 模型调用方式：OpenAI API
- 数据存储方式：本地向量索引
- 测试框架：pytest
- 不希望引入的依赖：重量级桌面 UI 框架

## 8. 安全与成本约束

- 不上传私人文档到不必要的外部服务
- 不在日志中记录 API key
- 默认使用低成本模型做索引相关操作
- 不修改用户原始文档
- 不执行不可逆操作
- 不批量删除索引文件

## 9. 验收标准

1. 能对指定文档目录完成检索问答
2. 答案能返回引用来源
3. 常见问题下结果稳定可复现

## 10. 当前优先级

1. 先做 CLI MVP
2. 再补检索质量
3. 最后再考虑 UI

## 11. 备注

- 这是一个示例项目，用来展示如何填写 `project_brief`
- 真实项目中技术栈和约束需要按实际情况替换
